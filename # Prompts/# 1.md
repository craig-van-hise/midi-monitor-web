**System Context & Objective:**
The current local repository is a Vite/React application that has already been pushed to GitHub. However, it is missing the GitHub Actions workflow required to actually build and deploy the app to GitHub Pages. Your task is to autonomously generate and push this missing CI/CD configuration.

**Task 1: Generate Deployment Workflow**
* Create the required directory structure at the root of the project: `.github/workflows/`
* Create a file named `deploy.yml` inside that directory.
* Populate `deploy.yml` with the standard GitHub Pages deployment action for a Node.js/Vite project. It must include:
  - Trigger on push to the `main` branch.
  - Required permissions (`contents: read`, `pages: write`, `id-token: write`).
  - Steps to: Checkout the repo, Setup Node.js (v20), run `npm ci`, run `npm run build`.
  - Deployment steps using `actions/configure-pages`, `actions/upload-pages-artifact` (targeting the `./dist` folder), and `actions/deploy-pages`.

**Task 2: Execute Git Commands**
* Execute the terminal commands to stage the new workflow: `git add .github/workflows/deploy.yml`
* Commit the file: `git commit -m "Add GitHub Pages deployment workflow"`
* Push the commit: `git push origin main`

**Task 3: Final Output**
* After successfully pushing the code, output a brief confirmation. 
* Remind the user of the final necessary step: They must navigate to their repository on GitHub -> **Settings** -> **Pages**, and change the **Build and deployment Source** dropdown to **GitHub Actions**.
