# Product Requirements Prompt (PRP) — ✅ Completed

## 1. Project Context & Objectives
You are tasked with expanding the visualization capabilities of the `midi-monitor-web` application. Currently, the "Continuous Data Stream" canvas exclusively plots continuous MIDI events (CC, Pitch Bend, Aftertouch). 

The objective is to introduce a multi-mode visualization architecture to this canvas. The user must be able to seamlessly toggle the graph between three modes:
1.  **Controllers** (Existing continuous data stream)
2.  **Notes** (A scrolling piano roll visualization showing note durations)
3.  **Velocity** (A vertical bar graph visualization representing the impact velocity of incoming notes)

## 2. Technical Decisions & Dependencies
To ensure ultra-high performance and prevent memory leaks inside the `requestAnimationFrame` loop, you will strictly adhere to the following architectural decisions:

* **Data Structure:** We cannot use the existing `streamsRef` for notes because notes have a duration (start time and end time). You will create a new high-performance reference: `notesRef.current = []`.
* **Type Definition:**
    ```typescript
    type ActiveNote = { 
      id: string; // e.g., `${channel}-${note}`
      ch: number; 
      note: number; 
      vel: number; 
      startT: number; 
      endT: number | null; 
    };
    ```
* **Memory Management:** You must actively prune `notesRef.current` during the render loop. Any note where `endT` is not `null` AND `endT < now - windowMs * 2` must be spliced out to prevent array bloating.
* **Color Mapping:** * **Notes:** Must use the existing `CHANNEL_COLORS[ch - 1]`.
    * **Velocity:** Must use an HSL heatmap ranging from Cool (Blue) to Hot (Red). Formula: `hsl(${240 + (vel / 127) * 120}, 100%, 50%)`.
* **Canvas Math:**
    * **Note Y-Axis:** `const y = h - (note.note / 127) * plotHeight - padding;`
    * **Note X-Axis:** Draw a line from `startX` to `endX`. If `endT` is null, `endX` equals the current `now` timestamp.
    * **Velocity Bar:** Draw a 2px vertical line at `startX` from `h - padding` up to `h - (note.vel / 127) * plotHeight - padding`.
