# Product Requirements Prompt (PRP) — Y-Axis Label Integration — 🔘 Pending

## 1. Project Context & Objectives
The multi-mode "Continuous Data Stream" canvas needs vertical Y-axis labels on the left-hand side to give users a clear quantitative reference for the plotted data, specifically targeting the 0-127 MIDI value range (which applies perfectly to Velocity, CCs, and Note Pitches). 

The goal is to dynamically reserve a left margin *inside the canvas drawing context* to render these guide numbers (127, 96, 64, 32, 0) without overlapping the active waveform/bar data, creating a clean, professional oscilloscope-style display.

## 2. Technical Decisions & Dependencies
Instead of relying on CSS flexbox or HTML DOM overlays (which can cause performance degradation and alignment issues on resize), you will render the text directly onto the HTML5 Canvas within the `requestAnimationFrame` loop.

**Required Canvas Math (Cognitive Offloading):**
* **Axis Boundary:** Reserve exactly 28 pixels on the left.
    `const axisWidth = 28;`
    `const plotW = w - axisWidth;`
* **X-Coordinate Translation:** ALL graph rendering across all three modes (Controllers, Notes, Velocity) MUST be updated to scale within `plotW` and offset by `axisWidth`.
    * *Formula:* `const x = axisWidth + ((timestamp - windowStart) / windowMs) * plotW;`
* **Text Coordinates:**
    * X: `axisWidth - 6` (Right-aligned against the graph edge)
    * Y: Corresponds to existing horizontal grid lines and padding bounds.
* **Colors:** Use `#555555` or `var(--color-text-dim)` equivalent for unobtrusive guide numbers.
